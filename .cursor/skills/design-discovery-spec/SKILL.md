# Design Discovery and Specification

## Instructions

1. Review PRD intent, user outcomes, and constraints.
2. Define key user journeys, edge cases, and error states.
3. Produce a UX/UI specification that engineering can implement.
4. Document accessibility requirements and interaction behaviors.
5. Highlight open design risks and unresolved decisions.

## Output Template

- Design scope and assumptions
- User journeys and state map
- UX/UI specification details
- Accessibility requirements
- Open risks and decisions needed
