# Engineering Execution

## Instructions

1. Confirm scope, constraints, and acceptance criteria.
2. Choose the simplest implementation that satisfies requirements.
3. Deliver in small, reviewable increments.
4. Keep architecture and naming consistent with existing code.
5. Add or update tests with each behavior change.

## Output Template

- Implementation approach
- Change list by increment
- Test strategy
- Risks/assumptions
- Done criteria
