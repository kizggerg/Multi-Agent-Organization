# Architecture and Security Review

## Instructions

1. Validate design against requirements and constraints.
2. Identify trust boundaries and threat scenarios.
3. Review controls for auth, data handling, and secrets.
4. Evaluate performance, reliability, and resiliency impact.
5. Document residual risks and required mitigations.

## Output Template

- Design assessment
- Security and NFR risks
- Required controls
- Residual risks
- Approval recommendation
